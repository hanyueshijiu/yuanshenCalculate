# 实现圣遗物计算器算法

## 算法思路

## 获取背包所有的圣遗物，首先进行筛选，是否选择+20？是否选择套装？ 对圣遗物进行格式化



## 圣遗物一共 5 个位置， 选择套装需要返回 4 + 1  这是最难的点，如果在上一步完全筛选了套装，这一步很难实现

## 如果每组圣遗物对象，添加一个计数器 count ，计算当前圣遗物套装的数量，在 count === 4 时 ， 第五个圣遗物从所有 套装 圣遗物中选择最优的

## 或者第一步，先实现一个 全套装的圣遗物计算器 ， 后续优化
